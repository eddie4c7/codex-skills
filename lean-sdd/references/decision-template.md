# Decisions Template

Use this template for `docs/decisions.md`. Record current active decisions and short reasons. Prefer this file over ADRs for ordinary choices.

```markdown
# Decisions

## Language

- Decision: [choice]
- Reason: [brief reason]
- Alternatives: [only if useful]
- Assumptions: [only if relevant]

## Framework

- Decision: [choice]
- Reason: [brief reason]

## Data Store

- Decision: [choice]
- Reason: [brief reason]

## Authentication

- Decision: [choice]
- Reason: [brief reason]

## State Management

- Decision: [choice]
- Reason: [brief reason]

## Test Strategy

- Decision: [choice]
- Reason: [brief reason]

## Deployment

- Decision: [choice]
- Reason: [brief reason]

## External Services

- Decision: [choice]
- Reason: [brief reason]

## Coding Conventions

- Decision: [choice]
- Reason: [brief reason]
```

If the user did not specify a technology, state the assumption and why the choice is sufficient for the current slice. Do not choose a heavier stack solely because future features might need it.
