# Product Definition Template

Use this template for `docs/product.md`. Keep it concise and product-focused. Omit sections that do not apply.

```markdown
# Product

## Problem

[What problem is being solved?]

## Target Users

[Who is this for?]

## Value

[What useful outcome does the product provide?]

## Main User Flows

- [User-visible flow]

## MVP Scope

- [Included capability]

## Out Of Scope For MVP

- [Explicit exclusion]

## Success Criteria

- [Observable product-level success signal]

## Assumptions

- [Assumption made to proceed]

## Unresolved Questions

- [Question that may affect future slices]
```

Do not include implementation details, framework choices, database schemas, or internal architecture in `docs/product.md`.
